## grep(查找)

### 基础参数

```shell
grep 参数 "查找内容" 文件名

参数
-i：忽略大小写
-v：反向查找，比如 grep -v "INFO" (我不看正常日志，只看异常)。
-n：显示行号。
-r：递归查找（查整个目录下的所有文件）。
-A / -B / -C (必考加分项)：
面试官问：“找到报错了，但我得知道报错前后的上下文，怎么看？”
grep -A 10 "Error" log.txt (After: 显示匹配行及后面10行)。
grep -B 10 "Error" log.txt (Before: 显示匹配行及前面10行)。
grep -C 3  "Error" 文件名（前后三行查看）
```

### 正则表达式

```shell
grep "^" 文件名	：匹配首行

```

### 管道搭配

```shell
ps -aux | grep nginx
cat 文件名 | grep 单词 | wc -l	#查看出现次数
```

### 找空行统计

```shell
grep -c "^$" 文件名
#会输出有多少空行
```



## sed **(主要用于替换)**

### 全句替换所有项

```shell
sed 's/旧内容/新内容/g' filename (替换显示，不改文件)。
```

### 替换每一行第一个匹配符

```shell
sed 's/cat/dog' 文件名
```

### 修改文件内容

```shell
sed -i 's/old/new/g' filename (-i 表示直接修改文件，面试官会问“怎么直接修改原文件？”)。
```

### 删除操作

```shell
sed '/^$/d' 文件名		#删除空行
sed '3,5d' 文件名		#删除特定行 3-5行
sed '/单词1/./单词2/d' 文件名	#删除从单词1到单词2之间的内容
```

### 添加/插入内容

```shell
追加内容（a命令）
sed '/指定词/ a 添加的内容' 文件名		#在指定行后添加
sed '/指定词/ a 添加的内容\' 文件名	#添加多行内容 \指换行符
sed '$a 追加内容' 文件		#$表示最后 a表示添加 在文本添加内容
```

```shell
插入内容（i命令）
sed '1i 插入的内容' 文件名	#1表示第一行 i表示插入 在指定行插入
sed '/word/i 插入的内容' 文件名	#在word前一行插入内容
sed '/word/i \\n' 文件名	#在word前插入空行
```



## awk**(主要用于取列/统计)**

```shell
场景： “请统计 nginx 日志里访问最多的 10 个 IP 地址。”（这道题出现率 90%）
原理： awk 默认按空格分割。$1 是第一列，$7 是第七列。

假设日志格式是：192.168.1.1 - - [Time] "GET /..."
命令：
awk '{print $1}' access.log | sort | uniq -c | sort -nr | head -n 10

awk '{print $1}'：取出第一列（也就是 IP）。
sort：排序（因为 uniq 只能处理相邻的重复）。
uniq -c：去重并计数 (count)。
sort -nr：按数字 (n) 倒序 (r) 排列。
head -n 10：取前 10 个。
```
